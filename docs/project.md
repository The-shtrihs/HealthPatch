## Слайд 3 — Один фіча в деталях: петля workout-to-PR

**👤 Максим · 0:30**

**🖼 На екрані:**

> ## Момент «кліку»
>
> ```
> 1. Почати сесію  →  вправи з шаблону копіюються автоматично
> 2. Залогувати підхід  →  повтори + вага збережено
> 3. Новий рекорд?  →  Personal Record автоматично upsert'иться
> 4. Доменна подія  →  PersonalRecordBeaten 🎉
> ```
>
> Все в **одній транзакції**. Інваріанти забезпечені value objects:
> `WeightKg`, `RepCount`, `SetNumber` — **від'ємна вага існувати не може.**

**🎤 Текст:**

> Ось момент, де гейміфікація «клікає». Користувач починає тренування, логує підхід з повторами та вагою — і система автоматично виявляє, чи це особистий рекорд. Все в одній транзакції бази даних. Фокус у тому, що «від'ємна вага» або «нуль повторів» не можуть існувати ніде в системі — вони блокуються на рівні конструктора value object, ще до того як будь-який рядок дійде до бази. Саме це робить решту коду безпечним для читання.

## Слайд 9 — Процес: гібрид Scrumban

**👤 Максим · 0:30**

**🖼 На екрані:**

> ## Чому чистий Scrum або Kanban нам не підійшов
>
> 4 людини, 4 різних домени → **3 паралельних міні-беклоги**, не один спільний.
>
> ## Що ми залишили
> - 🗓 **2-тижневі спринти** — ритм в умовах студентського розкладу
> - 🚦 **WIP-ліміт = 1** — закінчи перед тим як починати
> - 👀 **Pull-модель** — власники беруть завдання коли готові
> - 🪞 **Ретро кожні 2 тижні** — одна зміна процесу, занотована
>
> ❌ Без щоденних standup'ів. ❌ Без Sprint Review. ❌ Без виділеного Scrum Master.

**🎤 Текст:**

> Команда з чотирьох людей, де кожен власник різного домену, не має одного спільного беклогу — вона має три паралельних міні-беклоги. Чистий Scrum тут ламається. Чистий Kanban не має ритму, а команди без ритму — дрейфують. Тому ми зробили гібрид. Двотижневі спринти для каденції. WIP-ліміт один — закінчи розпочате, перш ніж братися за нове. Ретроспектива кожні два тижні з однією конкретною зміною процесу. Без щоденних standup'ів, без Scrum Master. Штучний двотижневий дедлайн — це те, що тримало нас у русі, коли навчальні дедлайни накопичувались.
